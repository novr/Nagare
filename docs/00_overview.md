# 00. プロダクト概要

## 1. プロダクトの目的

本プロダクト **Nagare** は、CI/CDプロセスを監視・評価するためのツール**ではない**。
開発チームが自らの開発フローの健全性をデータに基づき理解し、ボトルネックを発見し、具体的な改善アクションに繋げるための**診断・分析支援ツール**である。

Kent Beck氏の警告「指標が目標になると、それは良い指標ではなくなる」を核心的な思想とし、単一のスコアを追うのではなく、プロセスの全体像と傾向を把握することを目的とする。

## 2. ユーザーストーリー

### ペルソナ1：開発者

* `As a` 開発者,
* `I want to` CIの実行時間が長くなっている原因（特定の遅いジョブなど）を特定したい
* `so that` ビルドの待ち時間を短縮し、開発サイクルを高速化できるように

* `As a` 開発者,
* `I want to` プルリクエストのビルドが頻繁に失敗する原因（特定の不安定なテストなど）を知りたい
* `so that` 手戻りを減らし、`main`ブランチを常に健全な状態に保てるように

### ペルソナ2：開発責任者 / チームリーダー

* `As a` 開発責任者,
* `I want to` Four Keysのバランスと傾向を把握し、悪化している指標の根本原因をドリルダウンしたい
* `so that` データに基づいた改善アクションをチームと議論できるように

* `As a` 開発責任者,
* `I want to` 複数のリポジトリを横断した「プロジェクト」単位での健全性を把握したい
* `so that` 担当するサービス全体の開発プロセスが健全な方向に向かっているかを確認できるように
