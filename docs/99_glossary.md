# 99. 用語集

| 用語 | 説明 |
| :--- | :--- |
| **CI/CD** | **継続的インテグレーション/継続的デリバリー(デプロイメント)** の略。ソフトウェア開発において、ビルド、テスト、デプロイのプロセスを自動化し、迅速かつ頻繁なリリースを可能にする手法。 |
| **Four Keys** | Google CloudのDevOps Research and Assessment (DORA) チームによって提唱された、ソフトウェア開発チームのパフォーマンスを測るための4つの主要な指標。（デプロイの頻度、変更のリードタイム、変更障害率、平均修復時間） |
| **MTTR** | **Mean Time To Recovery (平均修復時間)** の略。Four Keysの一つ。本番環境で障害が発生してから、システムが完全に復旧するまでの平均時間。 |
| **ビルド (Build)** | ソースコードファイルを、実行可能な形式（ソフトウェア、アプリケーション、ライブラリなど）に変換するプロセス、またはそのプロセスによって生成された成果物。 |
| **プルリクエストビルド** | プルリクエストの作成や更新をトリガーとして実行されるビルド。コードをマージする前に、変更内容の静的解析やテストを自動実行し、品質を検証することを目的とする。 |
| **マージビルド** | `main` や `master` などの主要ブランチにコードがマージされたことをトリガーとして実行されるビルド。本番環境へのデプロイやリリース成果物の作成などを目的とすることが多い。 |
| **Apache Superset** | オープンソースのデータ探索・可視化プラットフォーム。本プロジェクトでは、ダッシュボード表示を行うWebアプリケーションとして利用する。 |
| **Apache Airflow** | オープンソースのワークフロー管理プラットフォーム。本プロジェクトでは、GitHubからデータを定期的に収集するバッチ処理（Data Collector）として利用する。 |
| **GitHub App** | GitHub APIと連携するための公式なアプリケーション形式。ユーザーアカウントやアクセストークンに代わり、特定の権限を持ってリポジトリにアクセスできる。セキュリティと管理性に優れる。 |
| **GitHub Actions** | GitHubに組み込まれたCI/CDプラットフォーム。本プロジェクトの主要なデータ分析対象。 |
| **DAG** | **Directed Acyclic Graph (有向非巡回グラフ)** の略。Airflowにおいて、一連のタスクの依存関係と実行順序を定義したもの。 |
| **マネージドサービス** | クラウドプロバイダーがサーバーの構築、運用、保守などを代行してくれるサービス。AWS FargateやAmazon RDSなどが該当する。 |
